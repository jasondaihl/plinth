# Plinth

A design system built on solid foundations. Tokens, components, and
documentation for building consistent, accessible UI at scale.

This Storybook is the working surface — every token, every component, every
state lives here.

## Three-layer tokens

Plinth tokens flow through three layers. Each layer is generated as a
separate CSS file; the brand layer registers tokens with Tailwind v4 so
utility classes like `bg-surface` resolve through the chain.

### Brand — Tailwind utilities

The brand layer surfaces semantic tokens to Tailwind v4's `@theme`, so any
consumer can write `bg-surface`, `text-text-muted`, or `border-border-strong`
and resolve to live token values.

<div className="flex gap-2 my-4">
  <button className="bg-interactive text-interactive-text px-4 py-2 rounded-md font-medium">
    bg-interactive
  </button>
  <button className="bg-surface-muted text-text border border-border px-4 py-2 rounded-md font-medium">
    bg-surface-muted
  </button>
</div>
